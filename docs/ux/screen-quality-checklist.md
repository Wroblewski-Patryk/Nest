# Screen Quality Checklist

- Is the primary action obvious within 5 seconds?
- Are `loading`, `empty`, `error`, and `success` states designed?
- Does the layout change meaningfully across desktop, tablet, and mobile?
- Is the touched flow comfortable for touch, pointer, and keyboard when
  relevant?
- Does the screen preserve Nest parity expectations across web and mobile?
- Is hierarchy created through spacing, scale, and contrast before color?
- Would this still feel premium after repeated daily use?
